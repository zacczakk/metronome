## What is Tableau?
Tableau is a Self-Serve Business Intelligence(BI) solution for Guided Analytics and Visualization that enable everyone to transform data into actionable insights that yield impact. With drag and drop, users can create interactive dashboards with advanced visual analytics.

- You can quickly perform ad hoc analyses that uncover hidden opportunities.
- You don't need to script any code.
- With drag and drop you can create interactive dashboards with advanced visual analytics.
- Tableau helps to make data driven decisions.

## What is UPTIMIZE Tableau?
UPTIMIZE Tableau is Merck's Tableau platform for Insights Generation and Self-Serve analytics. It is part of our UPTIMIZE eco-system and consists of three components - **UPTIMIZE Tableau Cloud,  UPTIMIZE Tableau GxP and UPTIMIZE Tableau On-Premise (aka UPTIMIZE MTableau).**
Our strategy is to have the gravity in the cloud and use On-Prem only if there are good reasons.

<br>If you read the name MTableau somewhere - MTableau is the predecessor of UPTIMIZE Tableau. 
The name might still be used while referring to our existing On-Premise Tableau Server.
However from now on everything will fall under the UPTIMIZE Brand so MTableau is not the official name of the platform anymore. 

<br>Tableau GxP hosts GxP-qualified Tableau dashboards and data sources, ensuring compliance and reliability for regulated environments.

## Difference between Tableau Cloud and On-Premise
With Tableau Cloud you can benefit from the latest updates by Tableau and the newest AI features which will only be made available in Tableau Cloud.
Our On-Prem version in contract will only be upgraded twice per year so you will get latest features much later.

## UPTIMIZE Tableau - Process from A to Z

You may have asked yourself - why do we need such a complex process?
<br>We hear and understand you that the beginning might be a bit hard but once everything is in place it will allow all Self-Service users to concentrate on the data and provide insights out of it for Merck.

<br>Let us first shortly explain, why we need the process as is. 
<br>In order to extend the value of Tableau across the entire organization and giving the **right balance between user flexibility and security/stability,** a clear governance concept needs to be in place.
<br>Tableau is based on a clear governance concept where every User Journey starts with a **Tableau Project.**

**A Tableau project is an environment to provide dashboards for others with same or similar purpose.**

You either need **access to an existing** or **having your own Tableau Project** in order to get a **Tableau license.**
By getting access to a project you will also getting a license - **every user needs to have a license.**
Today no costs are cross charged to the user or department cost center. Instead all licenses are distributed to our business sectors based on head counts.
To keep our license costs, low usage is monitored and unused license are being removed.

## Tableau Project - The structure on the Tableau Server to develop, deploy and share your Tableau Workbooks

Tableau User Journey starts with a Tableau Project. If you want to develop, deploy and share content on the Tableau Server you need a Tableau Project.
You can find detailed information on the Tableau Project folder structure below. If you want to learn more about Project Manager from a Project Owner or Deputy perspective please check the Project Management page and video.

A Tableau Project consists of multiple folders (see below) where you can <b> cluster multiple</b> workbooks (<b>dashboards</b>) for <b> different</b> user <b> groups</b> for the <b>same purpose</b> / use case lead by 1 project owner and up to 2 project deputies. 
Each project contains of:

Name and Description
Project Owner and Deputy
Set of Folders (Public, UAT, Workspace + special folder Exchange if required - see below for more details)
Set of User Groups (Creator, Explorer, Viewer - check Tableau Type of licenses below)
Permission and Authorization Structure (<b>Security</b> set on <b>Folders</b> and <b>NOT</b> on <b>Workbooks</b>)

## Project Folder - How many and what to consider

Each project has at least 1 public, 1 UAT and 1 workspace folder, but you can request as many public and workspace folders as you need. Each folder will have an own AD group to grant users access on. You can request new folders or delete folders anytime.

**Public**

* Request different public folders if you need to segregate content for different target groups and each group should not see the workbooks of the other group like Management, Sales Reps, Product Manager.

**Workspace** 

* Request different workspace folder
    * If you need to segregate different development teams like one team for Management and one for Sales Reps and each team don't allow to see the content of the others.

**Sub-Folders** 

* Subfolder within Public or workspace folder don't have any separate permissions....its only there to organize the content better

## License costs

Licenses are centrally managed by the Tableau Center of Competence Team. The team has the responsibility of managing the Demand with the Business Domain Leads and negotiate better pricing with Tableau for Procuring and Renewal as well.

Transparency on license usage is ensured through MTableau Cost Dashboard which will be used to review on a quarterly basis including details on license type usage per sector.

As stated above there are 3 License types that are available at Merck with the respective costs per Year listed below. 
The prices have been negotiated for Merck by the Procurement department of Merck.   

Creator:  467 Euro/ Year / User  
Explorer: 286 Euro/ Year / User  
Viewer: 112 Euro/ Year / User   

The Above costs include also an additional component called Server Management Add on that is charged by User by the Vendor. 

**Note - As stated at the beginning of this page there is NO individual cross-charging to user or department cost center.** 
Instead the license contract overall with all licenses costs is invoiced at the IT cost center and allocated back on business sector based on the global headcount key

Nothing is for free. There are costs associated for using Tableau depending on the license type associated to any user.

## Operational Framework - Time delay

There is <b> a time delay</b> in any request or change on the platform to provide operational stability on K2 as well as on the Tableau Server and Cloud.
<br>

### K2
It can take up to 2 hrs before the change is implemented and visible. Don't expect that right after you got an email that the request is approved and executed that the user is directly visible in K2.
<br>

### Tableau
It can take up to 24 hrs after the user request is approved and executed in K2 that the user can access the Tableau project.
<br>

### Developer (Creator)
Any request for access as creator needs to be approved by the Center of Competence. Here also some delay can happen before the request will be finally executed.

## Mail Bursting - User Guide

Mail Bursting allows you to automatically generate and distribute Tableau reports to specific users or groups. Instead of manually exporting and filtering, Bursting slices reports by **dimension** (e.g., Region, Segment, Category) and sends each slice only to the right recipient.

**1. When to Use Bursting**
* You want to distribute the same report but **filtered by e.g., role/region/category.**
* You need automated **scheduled delivery** to groups of people.
* You want to avoid **manual filtering/exporting** for each recipient.

**2. Accessing the Mail Burst Form**

i. From the left-hand menu, go to **Bursting → Create Mail Burst**.

<p align="center">
  <img src="assets/Tableau/tt.1.png" alt="Tableau Interface Screenshot" width="500" height="860"/>
</p>

ii. The **Add Mail Burst** form will open.

iii. Fill Out the Details.
- **Name:** Enter a name for your bursting task.
- **Status:** Active or inactive.
- **Source- Workbook Name:** Select the Tableau workbook to use.
- **Source - Workbook Views (optional):** Choose the relevant views of your workbook.
- **Recipients - Users:** Search for and add individual users.
- **Recipients - Groups:** Add Tableau user groups.
- **Output Settings:** Select file type (PDF, Excel), orientation (portrait, landscape), overlay (if needed).
- **Schedule:** Define when the burst executes (e.g., every Monday at 9 AM).

![tt](assets/Tableau/tt.2.png)

**Select the Workbook:**

![tt](assets/Tableau/tt.3.png)

**iv. Permissions Required**

To create or run a Mail Burst, you must have:

 Access to the **workbook** and its views.
 
Permission to distribute content to the selected users/groups. 

Mail Bursting permissions are controlled by your Tableau Tools role.
Only users with specific roles (e.g., Global/Cluster Administrator or Project Owner/Deputy) can create, edit, or execute bursts.

You can verify your access by checking the options available under **Bursting → Manage Bursting:**

-	If you can see the list of existing bursts, you have List permission.
-	If you can create a new burst, you have Create permission.
-	If you can open and edit burst details, you have Read/Edit permissions.
-	If you can run or delete a burst, you have Execute/Delete permissions.

If you do not see these options or receive a "Permission Denied" message, it means you do not have the required role or project assignment.

In that case, contact your Tableau Administrator or Burst Manager to review or assign the appropriate access level.

**Note:** Only administrators and designated project owners typically have full Mail Bursting control. Regular users can receive burst emails but cannot create or edit bursts.

## Notes & Limitations:

**Custom Views: Only public views can be used. Private user views are not supported.**

**XLS Output: Formatting may vary due to regional settings (decimal/date separators).**

**File Delivery: Reports are packaged in a ZIP file. Large bursts may generate large attachments.**

**Scheduling: Extract refreshes usually happen during early morning or off-peak hours (around 12:00 AM - 6:00 AM CET).**

**To avoid delays, schedule your Mail Burst after the refresh is completed, ideally 30-60 minutes later.**

**Avoid scheduling bursts during those times so reports run faster and always use the latest data**


## Business Role User Guide 

### Disclaimer

 - Please confirm with your **Tableau Platform Administrator** before using it.

###  Purpose
Business Roles help administrators **manage access easily** in Tableau Cloud.

Instead of giving each user access manually, you can create a role that includes: 
<br> A **license type** (Creator / Explorer / Viewer)
<br>The **projects or folders** the user of a respective role can access
<br>The **users** who belong to that role

This ensures consistency, saves time, and keeps access under control

### Creating a Business Role
Step 1: Open Business Roles<br>
<br>i. In the left-hand navigation menu,select **Business Roles → Create Business Role**<br>
<br>ii. The Create Business Role configuration screen will open.<br>

![tb](assets/Tableau/tb.1.png)

**Step 2: Define Role Details**<br>
<br>i. **Enter a descriptive name** for the business role.Example: LS PS - Business Analyst (Developer)
<br>ii. **Select the license type** to be associated with the role:
<br> Creator - for users who need to publish or edit dashboards.
<br> Explorer - for users who interact with published dashboards.
<br> Viewer - for users with view-only access.

![tb](assets/Tableau/tb.2.png)


**Step 3: Assign Tableau Projects or Folders**

* Access to all defined **projects/folders**

### Access Request Automation

When a user is added to a Business Role:-
 Tableau automatically creates individual access requests for each assigned project/folder.

**Example:**
 If the role includes **5 projects, the system creates 5 access requests** instantly -- no manual K2 submission required.

**6. Notes and Best Practices**

**Project Visibility:**
  <br>You, as the person defining a business role, can only define project access for projects you have access permissions to.

**Naming Convention:**
  <br>Use clear, consistent role names, including abbreviations for your sector, the department/initiative, the role and the license type in brackets e.g.: "LS PS - Business Analyst (Developer)"

**Review Regularly:**
  <br>Check Business Role memberships periodically to remove inactive users via K2. The business role helps to add users automatically, while the removal of users needs to happen manually by you in terms of a regular user review.

**Audit & Tracking:**
  <br>All role creation and assignment actions are logged for compliance.

# Tableau Extensions

 Extensions are add-on features that enhance Tableau's core functionality, helping users build more interactive and powerful dashboards.There are different sorts of extensions based on their usage and hosting environment:

* Viz Extensions and Dashboard Extensions (based on where they are used)

* Sandboxed Extensions and Network Extensions (based on how they are hosted)

For now, we are focusing on Sandboxed Extensions--these are more secure and easier to deploy since they run in a controlled environment within Tableau and don't require complex network setups or added costs.

With the latest version of Tableau Desktop, the following three extensions are enabled and ready to use:

[Sandboxed](https://docs.uptimize.merckgroup.com/tableau/tableau-extensions/sandboxed/) - Secure, self-contained extensions running within Tableau

[WriteBackExtreme](https://docs.uptimize.merckgroup.com/tableau/tableau-extensions/writeback-extreme/) - Allows users to write data back to the source from the dashboard

[SuperTables](https://docs.uptimize.merckgroup.com/tableau/tableau-extensions/super-tables/) - Adds advanced table features like inline editing, filtering, and grouping

# Tableau Sandboxed Extension

Tableau Sandboxed Extensions allow users to enhance their dashboards with additional functionalities. These extensions are designed to be secure and operate within a controlled environment, ensuring that they do not compromise the integrity of the Tableau platform. We support various types of extensions, including both free and paid options, with specific governance policies in place for Tableau creators, explorers, and viewers.

**Types of Extensions**
<br> We support all types of Sandboxed Extensions that are free to use without any blockers. However, if you choose a Sandboxed Extension with a paid version, you will need to raise a request for extensions as represented in the below Process flow diagram for Sandboxed Extensions.
  
For more information, you can visit [Tableau Extensions](https://exchange.tableau.com/extensions).

![sandbox](assets/Tableau/sandbox.png)

## How to Request Paid Sandboxed Extensions
To request paid Sandboxed Extensions, you need to use the request form that is available in Canny (Need to finalize) tool for pricing purposes.

**Approval Process**
<br>We only need to focus on the pricing and demand; there is no requirement to conduct any security or network analysis.


**For Explorers and Viewers**
<br>Explorers and viewers do not need to follow any of these steps. They simply need to ensure that they have access to the particular project. If you do not have access to the project where the dashboard is hosted, please raise a request through [K2](https://k2.merckgroup.com/Runtime/Runtime/Form/IT.Tableau.Forms.Dashboard/? ) and then you can directly access it from the servers.

## The Platform and the details

Tableau Server (check here how to logon https://docs.uptimize.merckgroup.com/tableau/getting-started-access-and-governance/login-to-server-and-cloud/ ) is the end-to-end data and analytics platform for Tableau. It helps connect people to trusted data and workbooks with insights. Developers can share their workbooks with users in Tableau Projects. Viewers get access to those workbooks and collaborate on the content shared.

The server is the central place where all Tableau Workbooks are published for users to get access. On the server you can:

* View and interact (filter, download, share, export) with content like dashboards and reports
* Collaborate with others using comments
* Create custom views on the workbooks (by pre-defining filters)
* Subscribe to receive the workbooks via email
* Create alerts and get notified when targets are met
* Publish your content as developer

# Tableau Reporting Styleguide

Standards are essential for several reasons. Firstly, they streamline the design and creation of reports, ensuring a more efficient process. Additionally, standards simplify the understanding of complex data, making it more accessible to users. They also define specific chart types for different data sets, which enhances clarity and consistency in data representation. Finally, by limiting variability across Merck, standards facilitate easier onboarding for new users, allowing them to adapt more quickly to the reporting processes. This results in a homogeneous reporting landscape, which further supports the reusability of templates for various tasks, maximizing efficiency and effectiveness in reporting practices.

## Why IBCS?

The main idea, "Same things look the same," emphasizes that the IBCS framework includes control areas grounded in scientific insights to optimize report construction for efficient information extraction. Aligning closely with IBCS offers several key advantages, including the ability to leverage proven best practices and avoid the trial-and-error process of developing company-specific standards from scratch, especially as IBCS is planned to be translated into ISO 24896, making early alignment beneficial. The IBCS standards are grounded in extensive research and industry experience, providing access to certified expertise and ensuring compatibility with widely-used tools like Tableau and Power BI, while also allowing for business-specific customization; this hybrid approach combines the reliability of a tested framework with the flexibility to meet Merck's unique requirements. 

All conventional tools, such as Tableau and SAC, can effectively implement the IBCS standard without hindering Merck's typical color theory. Ultimately, consistent adherence to the IBCS guidelines is more important than the specific choice of tool used for implementation.

## What these Guidelines are

*	Standardization of reports for faster and more effective understanding
*	Guidelines to help report designers to focus on content not format
*	Reference workbook and inspiration for report designers
*	Derived from but not limited to existing standards

## What these Guidelines are not

*	Rules to be followed dogmatically in all cases without exception
*	Governance rules which will be strictly enforced
*	Limitations for sectors to amend or adjust the guidelines to their needs

# UPTIMIZE Tableau GxP Platform

## Introduction
UPTIMIZE Tableau GxP platform is a new addition to the already existing UPTIMIZE MTableau platform. This new platform will be used by the users to create and publish Tableau workbooks and data sources which are specific to GxP use cases.

## Architecture
UPTIMIZE Tableau GxP platform consists of three environments Development, Validation \ QA and Production. Each environment consists of three nodes in which the Tableau Server application is installed and configured with an External File Store.

![GxP1](assets/Tableau/GxP1.png)

<center><b>High-level architecture diagram</b> </center>
<br>
UPTIMIZE Tableau GxP environment consists of three nodes, each node is a Windows Server in which the Tableau Server application is installed, and these Windows Servers are hosted in the internal data zone.

External File Store is a network file share location which is hosted in the confidential data zone and Tableau Server File Store stores extracts and workbook revisions.

<br>
Active Directory is used as the identity store and will be used to manage user and group information
<br>

[Tableau Tools](https://docs.uptimize.merckgroup.com/tableau/getting-started-access-and-governance/project-management-via-tableau-tools/) is a governance application which is used for content management and user access management. This is a non-GxP application which will use RESTAPI to connect to Tableau Server. 
For more details refer [here](https://docs.uptimize.merckgroup.com/tableau/getting-started-access-and-governance/project-management-via-tableau-tools/).

[K2](https://docs.uptimize.merckgroup.com/tableau/getting-started-access-and-governance/access-management-via-k2/) application is used by end users to request access to the Tableau Server, and this is a non-GxP application. 
For more details refer [here](https://docs.uptimize.merckgroup.com/tableau/getting-started-access-and-governance/access-management-via-k2/).
