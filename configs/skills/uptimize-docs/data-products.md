# Data Products
## What is a Data Product? 

A data product is a well-managed combination of high-quality data that enables the delivery of business value, meets user needs, and encourages reuse. It is easily findable and accessible via the Merck data catalog UPTIMIZE Data Central. A data product is part of a data product portfolio and is managed along a life cycle. 

Data Products characteristics: 
-	Findable*
-	Accessible*
-	Interoperable*
-	Reusable* 
-	Trustworthy
-	Self-describing
-	Owned

*FAIR principles 

Please also see the [respective glossary item in Data Central](https://datacentral.uptimize.merckgroup.com/explorer/glossary-items/765cea50-327b-4059-b8dc-caa58fa85d4a)

The concept of Data Products is agnostic to platforms, yet Data Products need platforms to exist. 
[Data Central](https://docs.uptimize.merckgroup.com/data-central/) is the capability of UPTMIZE to curate data products and make them accessible to the organization. 


## Why is the topic of Data Products important? 
Data provisioning at Merck has been historically driven by specific use cases with a focus on feasibility and less on value or re-usability, resulting in extra work and duplicated efforts in the long run. 
Treating Data as products brings the needed focus on ownership and data quality, improving the situation of issues with reliability of the data, ultimately with the goal to enable Analytics & AI at Merck even better & faster - seamless data, seamless experience. 

![Data-and-AI-ambition](https://docs.uptimize.merckgroup.com/docs-assets/data-product/Data-and-AI-Ambition.png)

## Who is responsible for Data Products? 
Ideally: Everyone! Data Products should be owned by Data Domains, which also means that there is no central organization that is intended to own all data products of the company. 
The MDAO and the Data Products Portfolio team own the Data Products concept, as well as the platform on which Merck brings Data Products to life: Data Central (https://docs.uptimize.merckgroup.com/data-central/) 
MDAO builds and offers own Data Products, and so do other organizations of Merck - be it the Sector Data Offices, or Spoke teams themselves.

# Guidelines for Filling Metadata for Data Products in Data Central

At Merck, we believe that data products are crafted with care, and we want that to shine through in the metadata you provide. Here are some friendly tips to help you fill it out effectively:

**1. Prepare Before You Start -**
Gather all necessary information about your data product before diving in. This preparation will make the process smoother and more efficient. Here's what to consider:
- Choose a straightforward and descriptive name for the data product: It should clearly convey its purpose and content.
  - **Good Example:** "Employee Details (Anonymized)"
  - **Bad Example:** "Data Dump 2024" (not very informative!)
- Identify where the data is coming from for accurate description.
- Identify key stakeholders involved in the data product for collaboration.
- Gather existing documentation or resources related to the data product.
- Identify any limitations or considerations users should keep in mind.

By gathering this information upfront, you'll be better prepared to create comprehensive and effective metadata for your data product.

**2. Check for Typos and Grammar -**
Always proofread your metadata description for typos and grammatical errors. A little attention to detail goes a long way! If you're unsure about anything, MyGPT is here to help -- just ask!

**3. Using Acronyms and Abbreviations -**
Use acronyms and abbreviations only if they are widely understood. 
For example, "HR" is clear, but if you're using "SCA," explain it as "Supply Chain Analytics (hereafter 'SCA')." 

**4. Crafting the Description -**
Provide a comprehensive overview of your data product:
- **Purpose:** Why does this data product exist?
- **Scope:** What areas does it cover?
- **Potential Use Cases:** How can it be used?
- **Target Audience:** Who will benefit from it?

To enhance clarity, incorporate visual aids, such as images, to explain the purpose and structure. Organize points using bullet points or numbered lists for easy reading. Don't forget to mention any limitations or considerations users should keep in mind!

**Just a quick note:** Some data is automatically retrieved from source systems like Foundry, so please remember to include a clear description in those sources as well!

**5. Linking Glossary Items -**
Identify and link relevant glossary items for any key terms that need definitions. This helps users understand the terminology without confusion.

**6. People Associated with the Data Product -**
Clearly identify the data owners and data stewards, along with their roles and responsibilities. Including their contact info or links to their profiles in the Contact field will facilitate easy communication.

**7. Describing Terms of Use -**
Specify any restrictions on data usage, such as prohibiting performance tracking of individuals in certain roles. Clear guidelines help ensure everyone uses the data ethically and responsibly.

---

## Quick Checklist:

✔ Gather necessary information before starting.  
✔ Proofread metadata for errors.  
✔ Use clear acronyms and abbreviations.  
✔ Provide a detailed description including purpose and audience.  
✔ Link relevant glossary items.  
✔ Identify data owners and stewards.  
✔ Specify terms of use and restrictions.

Following these guidelines will help you create metadata that reflects the quality of your data products!

## What categories of data products exist?
We use the differentiation & categorization into the following three types of data products:
- Source-aligned data products: data products that consist of data that has undergone minimal transformation.
- Consumer-aligned products: these data products consist of data, that has been prepared for a specific use case.
- Aggregate data products: a combination of various data products, for a specific use case - e.g. for KPIs at corporate level.

## Some examples from our UPTIMIZE world: 

- Source-aligned data products: Data is rather close to its "raw" state, but prepared for usage. Data Harbor, e.g., is the UPTIMIZE capability in which data from ERP systems is provided for a broad variety of use cases.
"Material Movements", "Purchase Orders", etc. would be examples in the sense of what Data Harbor shall offer in future.
- Consumer-aligned data products: A data set that combines Bill-of-Materials & inventories data, to enable production planning use cases 
- Aggregate data products: various data has been aggregated to be a basis for a KPI dashboard in Tableau or similar. 

(a clear distinction of consumer-aligned and aggregate data products is not always easy. Please understand the examples on this page as such -- examples.)

## Why is it important to categorize data products?
Understanding the distinctions helps organizations effectively manage their data. Source-aligned data ensures accuracy and reliability, consumer-aligned data enhances user experience by meeting specific needs, and aggregate data products provide comprehensive insights for informed decision-making. Each type serves a unique purpose, allowing businesses to leverage data more strategically. Recognizing that these three different categories of data products co-exist, also helps navigating the world of "what is a data product". In that sense: Flour that you take out of your storage is as much a product as the cake itself. Often cases you will look for flour, in case you want to do something else than cake.

# Data Products - Life Cycle and Portfolio Management 

## Just like every other product, a data product has a life cycle. 
For data products, the life cycle can be described in three phases: 
1)	Design Phase
2)	Consumption & Maintenance Phase
3)	Decommissioning Phase

![Data-Products-Life-Cycle.png](https://docs.uptimize.merckgroup.com/docs-assets/data-product/Data-Products-Life-Cycle.png)

## Data Products should be treated as part of a portfolio. 
Data Product Portfolio Management is the process of systematically selecting product ideas, continuously assessing, and optimizing the portfolio to maximize its value over time through alignment with organizational goals.
Reasons for managing a data product portfolio
(non-exhaustive): 
-	Strategic Alignment: Ensures that data products align with overall business strategy and goals, facilitating better decision-making and value creation.
-	Resource Optimization: Allows for better allocation and utilization of resources across various data products, minimizing waste and maximizing efficiency.
-	Risk Management: Enables identification and mitigation of risks associated with individual data products and the overall data domain, leading to informed risk assessment & effective mitigation.
-	Performance Monitoring: Provides systematic tracking of performance metrics across data products, allowing for comparative analysis and identification of underperforming assets.
-	Lifecycle Management: Facilitates effective management of the entire lifecycle of data products, ensuring they remain relevant and valuable from development to retirement.

## Data Product Cockpit Introduction

The Data Product Cockpit is an essential application designed for data producers, including portfolio and data owners, business data stewards, technical stewards, and data domain owners. At Merck, we define data products as a well-managed combination of high-quality data that delivers business value, meets user needs, and encourages reuse. These data products are findable, accessible, interoperable, and reusable. The central repository for Merck's data products is our Data Central Catalog. While this catalog is a powerful tool for searching and finding data products, it has limitations when it comes to creating and returning metrics about these products and their metadata. For example, metadata completeness of data products is only displayed on a single product level, aggregation on a sector or data domain level is not possible. 
To address these limitations, we developed the **Data Product Cockpit**, which offers a more granular and flexible approach to searching and creating metrics around data products. A reminder, that is not allowed to use the Data Product Cockpit and the metrics provided to track performance of individuals, for example with a data producer role.  

## Accessing the Data Product Cockpit

The Data Product Cockpit is currently housed in Uptimize Foundry as a Contour application. To access this Contour front end, you need to request access to the "Optimized Data Central Producer Metrics" use case for the Data Product Cockpit via the **Use Case Portal**. You will find the QR code link to the use case in the resource chapter. Once you have received access, you can open the Data Product Cockpit Contour application. The application can be found in the analysis folder in the use case project. To access it, you can follow the link in the use case description or by scanning the QR code provided in the resource chapter.

## Data Sources of Data Product Cockpit and Synchronization with Data Central Data Catalog

The data source for the Data Product Cockpit is Data Central. Currently, there is a connection for metadata related to data products, metric items, glossary items, data domains, and user adoption data from Data Central itself. This data is synchronized on a daily basis; however, due to pipeline latency, there is currently a time lag of approximately two days between the data visible in Data Central and the data displayed in the Data Product Cockpit.

# Navigating the Data Product Cockpit

## Central Elements of the Data Product Cockpit
The Data Product Cockpit user interface provides three main elements:

 1. **Central Display Area** - Located in the center-right, this area showcases all data, dashboards, graphs, and tables. Use the scroll bar to move up and down to access all contents.
 2. **Tab Ribbon** - Positioned at the top, this ribbon features different tabs that focus on various metadata topics of Data Central data. Currently, we cover the following topics:
   - **Overview Tab**: Provides an overview of the most important metrics.
   - **Data Products Tabs**: Offers metrics about Data Products in the Data Catalog and Marketplace of Data Central, including the historical development of key metrics.
   - **Data Domain and Roles Tabs**: Provides metrics and ownership information for data domains and data product portfolios.
   - **Glossary Item Tab**: Offers metrics about glossary items of Data Central, including the historical development of key metrics.
   - **Metric Item Tab**: Provides metrics about metric items of Data Central, including the historical development of key metrics.
   - **User Adoption Tabs**: Offers metrics about user adoption of the Data Central Data Catalog application, including the historical development of key metrics.
 In some cases, metric information, such as metadata completeness, is shown color-coded depending on the numeric value displayed, allowing for quick identification of metrics with room for optimization. In general, but not in all cases, an equidistant quartile scale is applied.
 3. **Custom Filter Parameter Window** - Found on the left, this collapsible window allows you to filter the displayed data based on custom parameters, such as selected sectors, data domains, names of individuals with specific roles, and, depending on the data, dates. It is possible to store preselected filters as links for frequent use.

## Custom Filtering in the Data Product Cockpit

The Data Product Cockpit offers powerful custom filtering options through a slide-out filter window. If the filter window is not visible on the left side, you can select the "X" in round brackets. In this window, you will see the following filtering parameters:
 - **Search in All Fields**: Enables searching across every available column of text or string type.
 - **Selected Sector**: Filters data by business sector.
 - **Selected Data Domain**: Allows filtering by specific data domains.
 - **Role Name**: Includes roles such as curator, data owner, domain owner, technical steward, and business steward.
 - **Item Name**: Filters by item names, such as data products or glossary items (depending on the respective tab).
 - **User Adoption Views**: Filters by selected departments and functional areas.
 - **Time Period**: For time series data, you can specify a start and end date to limit the results.
The filter board in the Data Product Cockpit offers two methods for entering the filter parameters:
- **Text-Based Search Fields** : For the parameters "Search in All Fields," "Role Name," and "Item Name," you can type in text to search. The search is conducted across the entire column and supports partial text matches. It also ignores capitalization, so whether you use uppercase or lowercase letters, the results will remain the same. You need to confirm your input with the return key.
- **Drop-Down and Calendar Inputs** : For fields such as "Sector," "Data Domain," "Selected Level 2 Department," and "Selected Functional Area," pre-populated drop-down lists are available. When you begin typing, matching values will appear for selection. For the "Time Period" fields, including start and end dates, a calendar interface allows you to pick the desired dates easily.

**Making Selections and Applying Filters**
- You can make multiple selections per filter; the selected values are displayed in a tile style in the entry field.
- When multiple parameters are entered in one specific filter, all results displayed in the dashboard will match at least one of the provided values. For example, if you enter two names in the "Roles" filter parameter, only assets linked to at least one of the role names will be displayed.
- If different filter parameters are activated in the filter window, only results that meet all activated filter requirements will be displayed in the dashboard.
- Filter parameters that are left empty or inactive will be ignored in the filtering process.
To make a filter selection effective, click the **Apply** button located at the bottom of the filter window.

**Reverting and Updating Filters**
- If you want to revert your filter selection, select the **Undo Arrow** icon on the top right of the filter window.
- Alternatively, you can delete single values by clicking the "X" in the value tile.
- To update a selection, click the **Apply** button again.

**Viewing Active Filters** - Once custom filters are applied, all tabs in the dashboard are updated accordingly. At the top of each tab, an overview of active filter parameters is provided. You can hide the filter window by selecting the **Slide In** icon in the top right of the filter window to gain space for the dashboards.

**Starting with Preselected Filters** - To start the Data Product Cockpit with an already preselected set of filters, click on the link symbol at the top of the filter window and save the copied link with the currently applied filters as a favorite in your browser.

These flexible filtering options help users tailor their searches to specific needs and contexts.

## Introduction

The [Data Product Cockpit Agent](https://mygpt-suite.uptimize.merckgroup.com/chat?a=698b76d2-268a-44ff-9fc3-4a1c2b26a845) is a conversational interface that makes insights from the Data Product Cockpit accessible through natural language. Instead of manually navigating dashboards, users such as portfolio and data owners, business and technical stewards, and data domain owners can simply ask questions like "Which data products in Healthcare have over 90% metadata completeness?" or "Which data products in the Application Data Domain do not have an assigned Business Data Steward?" and receive targeted answers.

Built on top of Merck's Data Product Cockpit, which complements the Data Central Catalog, the agent supports analysis of data product and metadata quality across portfolios, domains, and sectors going beyond what the catalog alone can provide.
The agent uses the same datasets that power the Data Product Cockpit as its knowledge base. These datasets are synchronized with the agent on a 24-hour basis, ensuring that insights remain closely aligned with the cockpit's latest data.

### Who can access the Data Product Cockpit Agent?

Access to the Data Product Cockpit Agent is tied to the existing [UPTIMIZE Data Central Data Producer Metrics (Data Product Cockpit)](https://palantir.mcloud.merckgroup.com/workspace/module/view/latest/ri.workshop.main.module.cfbd54d9-5f5e-4c67-a94d-37fc842812cc?pageId=shared&useCaseId=b7d69526-1869-456e-a2e4-5ff2c5b356c3) use case.

- You must first have access to the Data Product Cockpit use case.
- Once this access is confirmed, you will be added to the agent user group.
- At the moment, adding users to this agent user group is a manual process.
